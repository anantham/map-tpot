## Lessons Learned

### Messaging Workflow
- Chrome DMs send on Enter ‚Äî multiline messages get fragmented into separate bubbles
- Always send full messages via Beeper; Chrome only for initiating new chats with short openers ("hey [name]")
- Search Beeper by **display name** not handle (handles don't match Beeper chat names)
- Opening a DM in Chrome isn't enough for Beeper to sync ‚Äî must send at least one message
- New DM chats take 5-30 seconds to sync from Chrome to Beeper. Search by display name if handle search fails.
- For people you know on WhatsApp/other platforms, send there instead of cold Twitter DM (warmer)
- Check shared Beeper groups to gauge relationship warmth before drafting
- Read existing DM history before sending ‚Äî avoids duplicate outreach, reveals relationship context

### Message Framing
- For uncertain BLR locations, frame as "know anyone who'd be into this" not "are you free"
- For confirmed BLR + thematically relevant people, use direct "are you free this week"
- Reason about each person's specific relevance before drafting (recent tweets, mutuals, bio)
- Quote their bio/work back at them ‚Äî shows you actually looked
- Encourage superconnectors to spread the word to their circles
- For community organizers (saxenasaheb, rvivek), use "cross-pollinate communities" framing ‚Äî peer-to-peer energy, not cold outreach
- For existing contacts with history of sharing events (paraschopra), user may prefer to handle personally ‚Äî always check

### Lead Discovery
- When a profile/account 404s, **search Twitter** for the handle ‚Äî they may have changed it
- Chrome search is powerful for discovering new leads (e.g., searching "adityaoberai1" found his new handle AND @ritviii via a co-hosting tweet)
- Track keywords from profiles for a wider net search at the end (see Keyword Search Campaign below)
- Mining replies from event posts (matcha walk, AI meetups) surfaces warm BLR builder leads at scale
- "You might like" sidebar on Twitter profiles reveals network clusters ‚Äî Rob Haisfield appeared on uberstuber's sidebar, confirming the TfT overlap
- Twitter search `https://x.com/search?q=KEYWORD&src=typed_query&f=top` ‚Äî use `&f=user` for People tab

### Chrome Extension Reliability
- Chrome extension disconnects frequently across sessions. Fix: `/mcp` command to reconnect, or restart Chrome.
- After Chrome restart, old tab IDs become invalid. Always call `tabs_context_mcp` first.
- Persistent "No tab available" bug: extension lists tabs via `tabs_context_mcp` but can't interact with any tool (screenshot, navigate, type). Root cause unclear ‚Äî possibly tab group ID mismatch. Fix that worked before: close all tabs, open fresh x.com tab in new window.
- When Chrome is down, Beeper-only workflow still works for people who already have synced chats
